= Sleep Number BLE
:toc: macro
:toclevels: 3
:icons: font

A Home Assistant custom integration for local BLE control of Sleep Number 360 beds.
No cloud connection required -- communicates directly with the bed's BAM module over Bluetooth Low Energy.

toc::[]

== Features

* *Firmness Control* -- Set sleep number (0-100) independently for each side
* *Foundation Presets* -- Activate Flat, Zero G, Snore, Read, Watch TV, and Favorite positions per side
* *Bed Presence Detection* -- Binary sensors indicating occupancy per side (polled every 20 seconds)
* *Auto Discovery* -- Automatically discovers Sleep Number beds via BLE service UUID
* *ESPHome BLE Proxy Support* -- Works through ESPHome Bluetooth Proxy devices

== Requirements

* Home Assistant 2024.1+
* A Bluetooth adapter accessible to Home Assistant, either:
** A local Bluetooth adapter on the HA host, or
** An https://esphome.io/components/bluetooth_proxy.html[ESPHome Bluetooth Proxy] (recommended for range)
* Sleep Number 360 bed with BAM module (BLE/WiFi controller)

== Installation

=== Manual

. Copy the `custom_components/sleepnumber_ble/` directory to your Home Assistant `config/custom_components/` directory
. Restart Home Assistant
. The bed should be automatically discovered under *Settings -> Devices & Services*
. Click *Add* to configure

=== Manual Configuration

If auto-discovery doesn't find your bed:

. Go to *Settings -> Devices & Services -> Add Integration*
. Search for "Sleep Number BLE"
. Enter the BLE MAC address of your bed

TIP: The bed's MAC address can be found in the SleepIQ app under *Bed Settings -> About Bed*.

== Foundation Presets

[cols="1,3"]
|===
| Preset | Behavior

| Flat
| Lowers the selected side's head and shared feet to flat position

| Zero G
| Raises both heads and feet to zero gravity position (whole bed)

| Snore
| Raises the selected side's head slightly

| Read
| Adjusts both heads and feet for reading (whole bed)

| Watch TV
| Adjusts both heads and feet for watching TV (whole bed)

| Favorite
| Moves to saved favorite position (whole bed)
|===

NOTE: The bed has two independent head actuators (one per side) but shared foot actuators.
Presets that move the feet affect both sides. Only one head can be lowered at a time --
going from Zero G to Flat requires two separate Flat commands (one per side).

== How It Works

This integration communicates with the bed using the *MCR (Multi-Channel Radio)* binary protocol
over BLE. The protocol was reverse-engineered from the SleepIQ Android app.

* The bed's BAM module exposes a BLE GATT service (`ffffd1fd-388d-938b-344a-939d1f6efee0`)
* Commands are sent as MCR frames (sync bytes + 10-byte header + payload + CRC)
* The integration connects on-demand for each operation, then disconnects
* Presence is polled every 20 seconds; full status every 5 minutes

For full protocol details, see link:../PROTOCOL.md[PROTOCOL.md].

== Known Limitations

* *Presence detection delay* -- The bed's pressure-based presence sensor takes ~15-20 seconds to register changes. This is a hardware limitation, not a software issue.
* *No push notifications* -- The bed is purely request/response. All state must be polled.
* *Foundation preset state* -- The bed does not report its current preset position via BLE. Preset selectors are fire-and-forget.
* *Single connection* -- Only one BLE client can communicate with the bed at a time. The integration uses connection locking to prevent collisions.
* *ESPHome proxy quirk* -- The ESP-IDF BLE stack silently drops `write-without-response` packets. This integration uses `write-with-response` as a workaround.

== Tested Hardware

[cols="1,1"]
|===
| Field | Value

| Bed Model | I8, Flextop King, 360 FlexFit 2
| BAM Firmware | 0.4.1d9
| Hardware Rev | EVT3
| BLE Proxy | EverythingSmartHome EP1, Apollo MSR-1
|===

Other Sleep Number 360 beds with BAM modules should work, though the MCR address derivation
and function codes may vary. Older beds without BLE support are not compatible.

== Troubleshooting

=== Bed not discovered

* Ensure Bluetooth is enabled and a proxy/adapter is in range
* Check that no other device (phone, SleepIQ app) is connected to the bed via BLE
* The bed advertises its MAC address as its BLE name -- look for a device named like `64:db:a0:07:dd:02`

=== "Failed to read bed status"

* Verify the ESPHome BLE proxy has `active: true` in its `bluetooth_proxy` configuration
* Check that the proxy is within Bluetooth range of the bed
* Ensure no other BLE client is connected to the bed

=== Presence not updating

* The bed's pressure sensor needs ~15-20 seconds of sustained weight to register presence
* Presence is polled every 20 seconds, so updates may take up to 40 seconds total
